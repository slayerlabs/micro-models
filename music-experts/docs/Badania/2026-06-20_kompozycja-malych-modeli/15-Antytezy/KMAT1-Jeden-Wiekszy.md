# KMAT1 — Jeden większy model (steelman)

## Antyteza
Jeden model uczy się **wspólnej reprezentacji**, w której wiedza komponuje się wewnętrznie. „Mali poprawni osobno" **nie gwarantują poprawności w złożeniu** — na styku reprezentacje się nie zgadzają i wynik jest niespójny. Jeden model = jeden trening, jedna geometria, brak problemu odwzorowania między przestrzeniami.

## Granica
Większy model = drożej (compute, pamięć), brak modularności, trudniej dokładać domeny. Jeśli kontrakt wymusza wspólną przestrzeń — argument o utracie spójności słabnie.
